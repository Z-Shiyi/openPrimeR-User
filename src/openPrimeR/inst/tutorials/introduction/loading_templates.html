Loading template sequences

Let’s try to load the sequences from the following FASTA file containing germline cDNA from human immunoglobulin heavy chains :

fasta.file <- system.file("extdata", "IMGT_data", "templates", 
        "Homo_sapiens_IGH_functional_exon.fasta", package = "openPrimeR")
print(fasta.file)
## [1] "/TL/opt/R-3.4.0/library/openPrimeR/extdata/IMGT_data/templates/Homo_sapiens_IGH_functional_exon.fasta"

Please use an editor to get acquainted with the format of the file and then take a look at the template sequences using read_templates().

fasta.file <- system.file("extdata", "IMGT_data", "templates", 
        "Homo_sapiens_IGH_functional_exon.fasta", package = "openPrimeR")
# Load the template sequences from 'fasta.file'
# Load the template sequences from 'fasta.file'
read_templates(fasta.file)

As you can see, the templates were successfully loaded; Please explore the loaded Templates. You can change columns by using the top right and left arrows. Comparing the loaded templates with the original FASTA file, we can see that the metadata contained in the headers of the FASTA file were not annotated by read_templates().

To store the metadata correctly, you can use the hdr.structure and delim arguments of read_templates(), where hdr.structure provides a character vector of identifiers for information contained in the header that are separated by the single character delim. The keyword ‘GROUP’ can be used in hdr.structure to identify which group a template belongs to. Now, it’s your turn: Try to load the the template sequences with their annotated IGHV groups:

hdr.structure <- NULL
delim <- NULL
template.df <- read_templates(fasta.file, hdr.structure = hdr.structure, 
                               delim = delim)
hdr.structure <- c("ACCESSION", "GROUP", "SPECIES", "FUNCTION")
delim <- "|"
template.df <- read_templates(fasta.file, hdr.structure = hdr.structure, 
                                delim = "|")

You can verify that the template groups are available now via:

fasta.file <- system.file("extdata", "IMGT_data", "templates", 
        "Homo_sapiens_IGH_functional_exon.fasta", package = "openPrimeR")
hdr.structure <- c("ACCESSION", "GROUP", "SPECIES", "FUNCTION")
delim <- "|"
template.df <- read_templates(fasta.file, hdr.structure = hdr.structure, 
                                delim = "|")
print(template.df$Group)

Having annotated the templates with their corresponding genetic groups, we can move on to defining the binding region of the primers in the templates. In case of our immunological templates, we want to annotate the leader region for each sequence individually. For this purpose, we will use a FASTA file containing the leader sequence for every loaded template:

leader.fasta <- system.file("extdata", "IMGT_data", "templates", 
        "Homo_sapiens_IGH_functional_leader.fasta", package = "openPrimeR")
print(leader.fasta)

You may again want to take a look at the structure of the file before continung. Note that the entries in the provided file should match those of the FASTA file containing the template sequences and that the provided sequences should be subsequences of the loaded sequences.

Let’s try to define the binding region for forward primers in the templates using assign_binding_regions():

leader.fasta <- system.file("extdata", "IMGT_data", "templates", 
        "Homo_sapiens_IGH_functional_leader.fasta", package = "openPrimeR")
leader.fasta <- system.file("extdata", "IMGT_data", "templates", 
        "Homo_sapiens_IGH_functional_leader.fasta", package = "openPrimeR")
template.df <- assign_binding_regions(template.df, fw = leader.fasta, rev = NULL)

We can verify that the binding regions for forward primers were annotated successfully in the following way:

where Allowed_fw contains the sequence of the leader region and Allowed_Start_fw and Allowed_End_fw provide the closed interval of allowed binding positions in the templates for forward primers.

As a final step, we want to modify the binding region for forward primer such that the binding region includes the first position of the exon, which lies outside the leader. Hence, we need to extend the current binding region by one position. Let’s use update.individual.binding.regions for this. we have just defined by one position. This function just requires the input of the new binding interval relative to the previously defined one.

template.df <- openPrimeR:::update.individual.binding.regions(template.df, 
        c(-max(template.df$Allowed_End_fw), 0), NULL)
Note that we have chosen the maximum relative position for the start of the binding region to not affect any change, while we have extended the end of the binding region to the 0-th position, where 0 indicates the start of the exon.
